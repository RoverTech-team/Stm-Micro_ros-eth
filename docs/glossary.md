# Glossary

A list of technical terms and abbreviations used throughout this project.

| Term | Definition |
|---|---|
| <span id="cm7">`CM7`</span> | **Cortex-M7** — The high-performance primary core of the STM32H755. |
| <span id="cm4">`CM4`</span> | **Cortex-M4** — The auxiliary core of the STM32H755. |
| <span id="dds">`DDS`</span> | **Data Distribution Service** — A middleware standard for data-centric connectivity. |
| <span id="hsem">`HSEM`</span> | **Hardware Semaphore** — A hardware block used for synchronization between the CM7 and CM4 cores. |
| <span id="lwip">`LwIP`</span> | **Lightweight IP** — A small open-source TCP/IP stack used for embedded systems. |
| <span id="microros">`micro-ROS`</span> | An implementation of ROS 2 specifically for microcontrollers. |
| <span id="mpu">`MPU`</span> | **Memory Protection Unit** — Used to define memory regions and their attributes (e.g., cacheability). |
| <span id="rmw">`RMW`</span> | **ROS Middleware Interface** — The layer that abstracts the underlying DDS implementation in ROS 2. |
| <span id="tap">`TAP`</span> | **Network TAP** — A virtual network interface on the host used for simulation. |
| <span id="xrce-dds">`XRCE-DDS`</span> | **eXtremely Resource Constrained Environments DDS** — The protocol used by micro-ROS to communicate with a ROS 2 agent. |
